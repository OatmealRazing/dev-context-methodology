# Priorities

## Project Priority

Used for cross-project ranking in the suggestion algorithm.

| Level | Weight | Description |
|-------|--------|-------------|
| high | 300 | Critical path, blocks other work or has a deadline |
| medium | 200 | Important, should be progressed regularly |
| low | 100 | Nice to have, work on when capacity allows |

## Backlog Item Priority (MoSCoW)

Used within a project for sprint planning.

| Level | Description |
|-------|-------------|
| Must | Required for the current cycle to succeed |
| Should | Important and expected unless a trade-off is needed |
| Could | Desirable, included if capacity allows |
| Won't | Explicitly not doing this cycle, with documented reasoning |
