# Item Types

## feature

New functionality or capability. Has a default task contract.
Requires estimation. Counts toward velocity.

## bugfix

Defect in existing functionality. Has a default task contract.
Requires estimation. Counts toward velocity.

## release

Version tagging, changelog, and publishing. Has a default task contract.
Estimation optional. Does not count toward velocity.

## refactor

Structural improvement without behavior change. Has a default task contract.
Estimation optional. Does not count toward velocity.

## spike

Research or investigation with a timeboxed scope. No task contract.
Requires estimation (represents the timebox, not the outcome). Does not count toward velocity.
Output: findings document with go/no-go recommendation.

## chore

Maintenance, housekeeping, or process improvement. No task contract -- just meets Definition of Done.
Estimation optional. Does not count toward velocity.
Retrospective actions automatically create chore items.

## Estimation Scale

| Size | Meaning |
|------|---------|
| XS | Minutes. Trivial change, no decisions. |
| S | A few minutes to an hour. Straightforward, no unknowns. |
| M | A few hours. Clear approach, some decisions to make. |
| L | A full session or more. Significant scope or unknowns. |
| XL | Multiple sessions. Should probably be broken down further. |
