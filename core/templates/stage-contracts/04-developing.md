# Stage 04: Developing

## Inputs

| Source | Location | Why |
|--------|----------|-----|
| Planning summary | ./stages/02-planning.md | Architecture and implementation plan |
| Setup summary | ./stages/03-setup.md | Confirms tooling and scaffold ready |
| Definition of Done | ./definition-of-done.md | Quality standard for deliverables |
| Backlog | ./backlog.md | Prioritized work items |

## Process

1. Review Definition of Done and confirm it's current
2. Populate backlog from implementation plan if not already done
3. Refine items with acceptance criteria, estimates, MoSCoW priority
4. Create sprint (if 3+ ready items) or work backlog directly
5. Execute engineering flow per item: analysis, development, review, testing, integration, done
6. Log daily progress in sprint (if active) or project notes
7. At sprint completion: review outcomes, run retrospective, update velocity
8. Repeat until all Must/Should items are done

## Outputs

| Artifact | Location | Format |
|----------|----------|--------|
| Sprint records | ./sprints/sprint-NNN.md | Goals, items, logs, outcomes |
| Retrospectives | ./retrospectives/sprint-NNN-retro.md | Reflections, actions |
| Updated backlog | ./backlog.md | Remaining items re-prioritized |

## Gate — Exit Criteria

- [ ] All Must items are done
- [ ] All Should items are done or explicitly deferred to Won't with reasoning
- [ ] No active impediments
- [ ] Definition of Done met on all completed items
- [ ] Final retrospective completed (if sprints were used)
