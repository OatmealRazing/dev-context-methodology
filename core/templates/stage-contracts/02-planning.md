# Stage 02: Planning

## Inputs

| Source | Location | Why |
|--------|----------|-----|
| Idea summary | ./stages/01-idea.md | Defines what we're building and for whom |

## Process

1. Brainstorm approaches and evaluate trade-offs
2. Write a design document covering architecture and key decisions
3. Choose tech stack with rationale
4. Create an implementation plan with milestones
5. Define success criteria (how we know it's working)

## Outputs

| Artifact | Location | Format |
|----------|----------|--------|
| Planning summary | ./stages/02-planning.md | Design, stack, plan, success criteria |

## Gate — Exit Criteria

- [ ] Design document or architectural approach documented
- [ ] Tech stack chosen with rationale
- [ ] Implementation plan exists with milestones
- [ ] Success criteria defined
