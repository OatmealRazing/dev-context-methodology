# Stage 06: Maintaining

## Inputs

| Source | Location | Why |
|--------|----------|-----|
| Shipping summary | ./stages/05-shipping.md | Release version and channels |
| Backlog | ./backlog.md | Ongoing issues and feature requests |
| Definition of Done | ./definition-of-done.md | Quality standard for patches |

## Process

1. Track issues and bug reports
2. Monitor dependency health and apply updates
3. Collect user feedback
4. Ship patches and minor updates (follow sprint cycle for larger work)
5. Conduct periodic project health reviews

## Outputs

This is an ongoing stage with no single output artifact. Maintenance activity
is tracked through the backlog and sprint records.

## Gate — Exit Criteria

This stage has no exit gate. Projects remain in Maintaining until moved to
Paused or Archived.
